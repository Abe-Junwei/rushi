| 文件 | 字段 |

| `apps/desktop/package.json` | `"version": "0.1.8"` |
| `apps/desktop/src-tauri/tauri.conf.json` | `"version": "0.1.8"` |
| `apps/desktop/src-tauri/Cargo.toml` | `version = "0.1.8"`（若与 tauri 同步） |

提交示例：

```text

```

## C. 打 Release 候选包

### C.1 对外 DMG + OTA（推荐）

需环境变量：

- `TAURI_SIGNING_PRIVATE_KEY`
- 可选 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

```bash
RUSHI_RELEASE_BUNDLE=dmg bash scripts/v1-personal-release-build.sh
bash scripts/release-postbuild-verify.sh
```

产物：

- `apps/desktop/src-tauri/target/release/bundle/macos/*.app`
- `apps/desktop/src-tauri/target/release/bundle/dmg/*.dmg`

### C.2 无签名钥（仅手测 .app）

```bash
npm run desktop:build-app:local
bash scripts/release-postbuild-verify.sh
```

> 无私钥时 **不可** 作为对外 OTA 终态；手测通过后仍须用 C.1 重打 signed 包再 tag。

## E. Go 之后：Git tag + GitHub Release

### E.1 确认工作树

- 手测 **Go** 对应的 commit 已 push（含 0.1.8 bump + 所有补丁）。
- **Signed** DMG 由该 commit 构建（或与 tag 指向同一 SHA）。

### E.2 打 tag 并推送

```bash
git tag v0.1.8
git push origin v0.1.8
```

或使用 annotated tag：

```bash
git tag -a v0.1.8 -m "macOS v0.1.8 — CLN-066 closure, ASR download UX, online STT chip"
git push origin v0.1.8
```

### E.3 CI / Release 验证

```bash
gh run list --workflow=release.yml --limit 3
gh release view v0.1.8
```

**期望资产**（macOS 首发）：

- `rushi-desktop_0.1.8_aarch64.dmg`（+ `.sig` 若启用 updater）
- `latest.json` 指向 0.1.8
- 可选：Linux `.deb`（若 CI 矩阵包含）

### E.4 发布后 smoke（5 min）

1. 从 **GitHub Release DMG** 安装（勿用开发机构建的 .app）。
2. 关于页版本 **0.1.8**。
3. 打开已有项目或新建 → 导入 → 波形 OK。
