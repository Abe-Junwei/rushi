## 1. 问题陈述

| 项 | 内容 |
|----|------|
| **用户场景** | 个人单机 v1.1 已交付主路径（R9 / REL-1.1 ✅）后，用户仍感知：**193 段级转写** 点选/滚动发涩、**转写后内存 ~3GB** 拖慢 WebView、**长音频编辑** scroll 层间偶发 jitter；同时有 **段旁标注**、**Welcome 跨文件搜索** 等产品债。 |
| **本仓现状** | **编辑器**：SCB Phase 0–1 已编码（store + imperative + reconcile 补丁）；`listCommit` ~400ms 根因仍在虚拟窗 pin / Orchestrator props（见 SCB plan Phase 2–3）。**波形**：tier scroll + sticky + `translate3d` 镜像（[`waveform-unified-scroll-stage-research.md`](./waveform-unified-scroll-stage-research.md)）；`MAX_WAVESURFER_PEAK_COLUMNS=40960` 已缓解 **canvas DOM 爆炸**（非 WebGL 动机）。**侧车**：ASR unload 薄片编码中（D8）。**数据**：`segments` state+ref 双轨（[`state-ref-convergence-research.md`](./state-ref-convergence-research.md)）。**产品**：segment-annotation / welcome-search 已有专项 research，未编码。 |
| **成功标准** | 本文输出：**≥8 条候选薄片** 的业内对照 + 可复用评估 + **推荐排期**；明确 **不做什么**（含 WebGL）；每条候选可在 **≤2 周单人** 内独立验收；组合路线不引入第二套 segment/selection 真源。 |

### 1.1 显式排除（本轮 portfolio 不推）

| 排除项 | 理由 |
|--------|------|
| **WebGL / WebGPU 波形** | zoom 非用户痛点；40960 cap + lazy canvas 已控 DOM；工程 1–2 月，ROI 低 |
| **Zoom LOD 增档 / drawPxPerSec 双轨** | 用户确认 zoom 不是问题 |
| **L1-C 全动态虚拟列表** | 5–8d，仅当 3000+ 段 + 长文听打成卖点 |
| **命令面板 ⌘K** | [`editor-keyboard-shortcuts-research.md`](./editor-keyboard-shortcuts-research.md) 已拍板不做 |
| **协作 / CAT / RAG / 实时 mic** | 路线图 §8 非 v1 |

## 3. 分域调研（≥2 条业内路线 / 域）

### 3.1 编辑器选中与列表 perf（P1–P3）

| # | 路线 | 代表 | 机制 | 链接 |
|---|------|------|------|------|
| A | **External store + row subscription** | TanStack Virtual、Virtuoso | 父列表不因 primary selection reconcile；行级 `useSyncExternalStore` | [TanStack Virtual](https://tanstack.com/virtual/latest) |
| B | **Imperative decoration** | Monaco | O(1) class 切换，不全树 re-render | [Monaco decorations](https://microsoft.github.io/monaco-editor/docs.html) |
| C | **Display/Interaction 分离** | Rushi B15 | Canvas 画全体；DOM 仅交互子集 | [`segment-overlay-virtualization.md`](./segment-overlay-virtualization.md) |

**评估**

| 路线 | 复用度 | Rushi 落点 | 冲突 |
|------|--------|------------|------|
| A | **高** | SCB-2/3 + 收敛后删 list imperative | 须先解耦 virtualWindow pin |
| B | 中 | 当前 SCB 列表分支 | 双写 reconcile 长期不可维护 |
| C | **高** | P3：`MAX_DOM_OVERLAY_SPARSE=32` 已存在；contiguous 须 cap | 连续全选 currently bypass cap |

**本仓可复用**：`selectionChromeStore` · `useSegmentRowSelection` · `useEditorSegmentListScroll` · `segmentListVirtualWindowCore` · `__rushiSelectionProfile`

**决策**：**P1 → P2 → P3** 顺序；P2 完成后 list imperative 仅保留波形 band/overlay。

---

### 3.2 内存与侧车（P4）

| # | 路线 | 代表 | 机制 |
|---|------|------|------|
| A | **显式 model unload** | Epicenter、whisper.cpp server | `unload()` + gc；空闲计时 |
| B | **进程 idle stop** | Rushi ASR-WARM | 杀 bundled 侧车 |
| C | **云 offload** | Descript Web | ML 不在客户端 |

**评估**：A+B **互补**（unload=RAM，stop=进程）；C 与产品定位冲突。

**真源**：[`asr-model-unload-on-file-switch-research.md`](./asr-model-unload-on-file-switch-research.md) — **已采纳**，编码中。

---

### 3.3 波形 scroll 架构（P5–P7）

| # | 路线 | 代表 | 机制 |
|---|------|------|------|
| A | **单 scrollContainer + renderer lazy** | WaveSurfer v7 | 内容随原生 scroll；canvas 分块 |
| B | **frameOffset + 命令式 canvas** | Peaks.js | 无 browser scroll 驱动画布 |
| C | **sticky + transform 镜像** | Rushi 现行 | tier 真源 + translate3d 补偿 |

**评估**

| 路线 | 复用度 | 说明 |
|------|--------|------|
| A | **中–高** | P5 目标形态；保留 tier 真源，去 C |
| B | 中 | 与 WS-only 引擎冲突大 |
| C | — | **技术债**；S1/S2 已优化但未根除 |

**DOM 爆炸**：现行 `clampPxPerSecForWaveSurferRender` + `40960` 列 cap → WS canvas **≤6 槽 / ~3 DOM**；**P5 动机是 scroll 顺滑与架构简化，不是防爆**。

**真源**：[`waveform-unified-scroll-stage-research.md`](./waveform-unified-scroll-stage-research.md) + plan/acceptance 已存在。

**决策**：P5 用 **1d spike**（去 mirror 原型 + 手测矩阵）再定是否全量；P6/P7 可与 P5 并行或作间隙填充。

### 3.4 数据层与架构债（P8、P12）

## 4. 组合决策与推荐排期

### 4.1 三轨并行（单人默认）

```text
轨 A · 编辑器 perf（连续 3–4 轮）
  P4 D8 unload          ← 收尾 v0.1.8.1（与 A 轨可交错）
  P1 SCB-2              ← 虚拟窗解耦（最大 listCommit 收益）
  P3 OVERLAY-CAP        ← 可与 P1 同轮或紧随其后
  P2 SCB 收敛           ← 删 list imperative，保留 reconcile 最小集

轨 B · 波形体验（1 轮 spike + 条件全量）
  P6 MINIMAP-CACHE      ← 间隙 0.5d
  P5 SCROLL-STAGE spike ← 1d go/no-go
  P5 全量 OR 维持 S2    ← spike 结论
  P7 PEAKS-WORKER       ← 可选，非阻塞

轨 C · 产品 / 架构（与 A/B 交错，勿与 CSP 大改同 PR）
  P10 SEG-ANNOT
  P11 WELCOME-SEARCH
  P8 STATE-REF S2.1–S2.2
  P9 L1-B（门禁触发）
```

### 4.2 不做什么（portfolio 级）

- 不把 SCB 双真源当作终态；P2 后 **SC2 imperative 仅限波形**  
- 不为 scroll 顺滑上 WebGL  
- 不在一条 PR 混合 P5 scroll 重构 + P1 SCB-2（回归面过大）  
- 不新开 segment/selection 第三套内核

## 8. 国内外产品横向调研（2026-06 扩展）

> **目的**：在 §2–§3 技术路线之外，对照 **可感知竞品** 的编辑器形态、perf 策略与功能边界，校验 portfolio 薄片是否对齐用户心智、是否有过度工程。  
> **方法**：官网/帮助中心/公开 changelog + 本仓已有 research（[`rushi-core-links-industry-comparison.md`](./rushi-core-links-industry-comparison.md) · [`segment-edit-stage-indicator-research.md`](./segment-edit-stage-indicator-research.md) · [`waveform-list-ux-boundaries-research.md`](./waveform-list-ux-boundaries-research.md)）+ 2026-06  web 复核。  
> **不做什么**：闭源实现细节臆测；把云产品架构原样搬到单机 Tauri。

### 8.1 产品分型（四象限）

```text
                    波形 / 时间轴权重
                           ↑ 高
         剪映 / Trint / Premiere          Descript（文稿+视频一体）
                           │
    云 ASR + 协作 ─────────┼───────── 本地 ASR + 离线
                           │
         飞书妙记 / 通义听悟 / Otter     MacWhisper / Rushi
         讯飞听见（会议闭环）              （口述史单机工作站）
                           ↓ 低
                    纯文稿 / 摘要为主
```

| 象限 | 代表 | 编辑心智 | 与 Rushi 关系 |
|------|------|----------|---------------|
| **本地听打台** | MacWhisper、Buzz、听见桌面（部分） | 音频 + 分段文稿；波形可选/弱 | **直接对标** |
| **云会议纪** | Otter、飞书妙记、通义听悟 | 摘要 > 精修；协作/评论 | 摘 AI 纪要 UX，不做实时 bot |
| **文稿驱动剪辑** | Descript、Trint | 改字 = 改时间线 | 借鉴双视图，不做 Overdub |
| **视频字幕工** | 剪映、Happy Scribe、Sonix | 视频轨 + 字幕块 | 借鉴 seek/sync，不做 NLE |
| **学术标注** | ELAN、Transana | 多层 tier + 时间对齐 | P10 标注真源 |

### 8.6 竞品路线对 portfolio 排期的校正

| 原排期 | 竞品启发后的调整 |
