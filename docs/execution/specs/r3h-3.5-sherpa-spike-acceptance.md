## 目标

验证 **Sherpa-ONNX Paraformer** 能否在可接受精度与语段质量下，作为 FunASR Python 侧车（~2.5GB）的 **轻量或中期替代候选**；**不以 SenseVoice 为 Spike 主模型**（已弃用）。

## 范围

### 做

| # | 交付 |
|---|------|
| 1 | Rust spike harness：`sherpa-onnx` + **P0** `paraformer-zh-2023-09-14` 离线转写 |
| 2 | **P1** INT8 变体体积/RTF 对比 |
| 3 | 同素材 **CER** + **13min 语段** 对比 FunASR 侧车（默认 Paraformer SKU） |
| 4 | 能力差距表：VAD、punc、**timestamps**、**hotwords**、平台 provider |
| 5 | macOS **CoreML**、Windows **CUDA/DirectML** 各 ≥1 smoke |
| 6 | 结论 ADR：`Go` / `Partial Go` / `No-go` |

### 不做

- 删除 `services/asr` 或 FunASR `prepare` 主路径
- SenseVoice / Qwen3 作为 Go 门控
- 环境页「引擎切换」产品 UI（Go 后另薄片）
- faster-whisper、whisper.cpp 全量评估
