## 1. 问题陈述

| 项 | 内容 |
|----|------|
| **用户场景** | 长音频（30min–3h+）转写后编辑：波形滚动、缩放、播放跟随、segment 拖拽、列表滚动须保持 60fps；ASR 侧车换文件后须释放内存但不牺牲再转写速度。 |
| **本仓现状** | v0.1.8.1 已完成 ASR 侧车 unload、PeakCache dispose、Hub/Welcome short-circuit、unified scroll stage S1–S5 自动验证；剩余 T1/T2 手测矩阵、PeakCache 拆分警告、minimap/segment overlay/zoom 等 waveform-optimization 项未实施。 |
| **核心约束** | **非 WebGL**（`waveform-optimization-proposal-2026-05` §8.3 L1 已明确 defer）；不替换 WaveSurfer.js v7；不迁移 Peaks.js；不默认切换 ASR 引擎；不引入第二套 scroll/选中真源。 |
| **成功标准** | v0.2+ 每个薄片的 ROI 可量化：滚动/缩放掉帧 ≤1 帧/60s、侧车 reload 增量 ≤+8s、手测矩阵全绿、自动闸门不回归。 |

### 1.1 当前关键链路

```text
Rust peaks 生成（2/20/200/800 pps）
  → Tauri command → useWaveformPeaks → PeakCache（内存 + resample LRU）
    → useProjectWaveformMount（WS v7）
      → useWaveformZoomSync（ws.zoom / ws.load 量子切换）
        → EditorWaveformPane
          ├─ tierScrollRef（scroll 真源）
          ├─ WaveSurfer container（Canvas2D tile）
          ├─ WaveformSegmentOverlay（DOM，虚拟化）
          ├─ WaveformSegmentBandCanvas（Canvas 色带，索引窗）
          ├─ WaveformLiveTimeRuler（时间尺 + playhead）
          └─ WaveformMinimapStrip（独立 Canvas）
```

## 2. 业内成熟路线（≥2）

| # | 路线 | 代表实现 / 产品 | 核心机制 | 与 Rushi 的映射 |
|---|------|-----------------|----------|----------------|
| **A** | **WaveSurfer.js v7 原生 content-tile + peaks 预注入** | WaveSurfer.js v7 官方 renderer | `scrollWidth = ceil(duration * minPxPerSec)`；单个 `scrollContainer`；renderer 按可见范围懒创建/复用 canvas；progress 不重绘波形；推荐预生成 peaks | 已部分对齐：预生成 `.dat`、注入 peaks、`autoScroll: false`、tier 真源。差距在于 DOM 舞台仍混合 sticky/mirror 遗留结构，以及 `ws.load()` 换档。 |
| **B** | **BBC Peaks.js 双视图 + 逻辑 frameOffset** | BBC Peaks.js（Konva + waveform-data） | `zoomview` 与 `overview` 分离；`frameOffset` 是视口起点；scroll 改 offset 后 canvas 重绘；segment marker 用 Konva/Canvas | 项目内已评估为「不推荐作为第一刀」（`waveform-unified-scroll-stage-research.md`）：需大改 stage DOM，与 WS-only 引擎冲突。仅参考其「单一 offset 真源」思想。 |
| **C** | **原生 DAW：时间坐标 viewport + 自绘波形** | Audacity / Ableton / Pro Tools | `hpos` 表示可见区左缘时间；播放头可固定，track 相对 hpos 重绘；summary/mipmap 保证长音频性能；sample-level 渲染无 LOD | 理念对齐：统一时间坐标、固定 viewport chrome、播放头与时间轴锁定。Rushi 因 Web/WebView 约束无法直接采用原生 sample 渲染，但可在 Canvas2D 内逼近。 |
| **D** | **Tauri + Rust 原生音频引擎 + WebView UI** | Ryosuke DAW 博客、tauri-sidecar-manager、Vox/AudioX/MumbleFlow | Rust 后端用 `cpal`/`symphonia` 播放并向前端 emit 波形数据；前端只负责可视化；sidecar 生命周期由 Tauri 管理 | 当前 ASR 侧车架构已类似（Python sidecar + HTTP），但播放仍走 WS 的 `<audio>`。此路线可用于未来把播放时钟迁到 Rust，但 v0.2 风险过高。 |

## 3. 可复用评估

### 3.1 路线对比

| 路线 | 复用度 | 可直接用的部分 | 与 Rushi 约束冲突 | 进度 / 内存 / 运维 |
|------|--------|----------------|-------------------|---------------------|
| **A WS v7 原生 content-tile** | **高** | peaks 预注入、`autoScroll: false`、`minPxPerSec`、progress 不重绘、renderer 虚拟化 | 当前 DOM 舞台 sticky + mirror 包袱与 WS 原生 scrollContainer 不完全一致；需要渐进收敛而非整体迁移 | 低进度风险；不增加内存；不引入新依赖 |
| **B Peaks.js frameOffset** | 低 | 「单一 offset 真源」思想；segment marker 管理 | 须替换 WS 为 Konva/Peaks 双视图；与 ADR-0005「WS-only」及已完成的 unified scroll stage 投资冲突 | 高迁移成本；新增 Konva 依赖；segment overlay 需重写 |
| **C 原生 DAW viewport** | 中（理念） | 时间坐标真源、固定 viewport chrome、播放头锁定、summary/mipmap | 无法直接采用原生 sample 渲染；WebView 内须用 Canvas2D/WebGL 模拟 | 理念已部分落地（tier 真源、projection）；纯 native 路径当前不可行 |
| **D Rust 原生音频引擎** | 低 | sidecar 生命周期管理、Tauri emit 模式 | 播放后端从 `<audio>` 迁到 Rust 是 v0.2 范围外大改动；与 WS playback 深度集成冲突 | 高风险；长周期；需音频流式传输设计 |
| **E sidecar 资源策略** | **高** | unload API 已存在；warm-stop 已存在；磁盘缓存（D4）已存在 | 侧车自我管理 idle timer 会与前端 Close Gate / busy 状态产生竞态 | 低风险；用户可见；自动化可验证 |

必须先列再决定是否扩展：

- `PeakCache` + `ByteBudgetLruMap`：已有 dispose/clear/LRU，可扩展 Worker/LOD/异步流式。
- `tierScrollFrameCoordinator`：已有单 rAF chrome repaint 协调器。
- `waveformProjection.ts`：时间↔像素真源。
- `useWaveformZoomSync` / `waveformZoomSyncEngine.ts`：8 px/s quantum、load 决策、双轨雏形。
- `WaveformSegmentBandCanvas` + `drawWaveformSegmentBands.ts`：索引窗 Canvas 色带。
- `WaveformSegmentOverlay` + 虚拟化：display/interaction 分离已完成。

- `useAsrModelUnloadOnFileSwitch`：前端触发 unload + 3s idle delay + 批量转写 skip。
- `smoke-asr-sidecar-health.sh`：已有 RSS soft WARN。

## 4. 决策摘要

### 4.1 总体方向

v0.2+ 在 **非 WebGL、保留 WS v7、保留 Python ASR sidecar** 的前提下，走 **「Canvas2D 极限优化 + 数据层解耦 + sidecar 资源策略细化」** 的组合路线。不谋求渲染引擎替换，而是把现有 WS 路径的剩余性能毛刺和架构债务清掉。

### 4.2 推荐薄片组合（按依赖顺序）

| 优先级 | 薄片 | 目标问题 | 关键动作 | 估时 |
|--------|------|----------|----------|------|
| **P0** | **PeakCache.ts 拆分 + resample 异步化** | guard 305 行警告；大音频 resample 阻塞主线程 | 拆 `ByteBudgetLruMap` 到独立文件；拆 level loader；把大数组 resample 改成 `requestIdleCallback` / microtask chunking | 2–3d |
| **P0** | **Minimap debounce + 离屏缓存** | resize 连续重绘 | RO debounce 100ms；peaks 不变时缓存离屏 canvas；resize 只做 `drawImage` 缩放 | 1–2d |
| **P1** | **Zoom 双轨去 `ws.load()`（完成 M2）** | 跨 quantum 闪白、拖动卡顿 | `drawPxPerSec` / `layoutPxPerSec` 双轨；拖动期只改 layout + `ws.zoom()`；pointerup/debounce 后必要时 `ws.load()` | 3–5d |
| **P1** | **Segment overlay 父级 re-render 优化** | 拖拽/滚动时父级 re-render | `laneByIndex` memo + signature 缓存；拖拽 draft 只更新对应 item；overscan 扩展 | 2–3d |
| **P1** | **播放跟随 rAF 化（完成 S1）** | timeupdate 250ms 跳跃 | `requestAnimationFrame` + `ws.getCurrentTime()` 驱动 ruler/playhead；scroll follow 接入同一 rAF | 1–2d |
| **P2** | **Segment list 动态行高 L1-B** | 长文语段非选中行仍裁切 | stride = max(设置行高, p90 测量行高)；仅可见窗 + pin 行 RO 采样；`rowHeightCache` + prefix-sum | 3–5d |
| **P2** | **ASR sidecar 资源策略细化** | 批量/手动/度量缺失 | 环境页「释放内存」按钮；批量转写期间已 skip，补充 queue 完成后自动 unload；smoke RSS 转 hard gate；侧车 idle timer spike | 2–3d |
| **P3** | **PeakCache Web Worker** | 主线程 resample 仍可能掉帧 | 大数组 resample + Float32Array 转换移入 Worker；小数据走 sync 短路 | 3–5d |

### 4.3 明确不做

| 项 | 原因 |
|----|------|
| **WebGL / WebGPU 波形渲染器** | `waveform-optimization-proposal-2026-05` §8.3 L1 已明确为长期技术储备，当前 defer |
| **Peaks.js / Konva 迁移** | `waveform-unified-scroll-stage-research.md` 已评估为「不推荐作为第一刀」；与 WS-only 投资冲突 |
| **自研 content-tile canvas peaks（ADR-0004 路径）** | 已废弃；统一回 WS-only |
| **Whisper.cpp / Sherpa ONNX 默认替换 FunASR** | `ADR-0006`/`ADR-0007` 门控未过；Sherpa 为 Defer |
| **Rust 原生音频播放引擎** | 播放后端迁移是高风险大改动，超出 v0.2 薄片范围 |
| **Segment list L1-C 全动态行高** | `segment-list-virtual-scroll-upgrade-options.md` 明确 Phase 0/1 不做；避免与波形薄片抢带宽 |
| **取消 160 pin cap（L3-D）** | 同上，5000+ 段 DOM 爆炸风险 |

### 4.4 与 ADR / Architecture 关系

- 继承 `ADR-0005`（superseded 但 tier 真源约束仍有效）、`desktop-waveform-engine.md`、unified scroll stage 成果。
- 不推翻既有坐标/播放模型，只优化实现层。
- ASR 侧车继续遵循 `ADR-0001` HTTP 契约 + D4/D8 正交设计。

### 4.5 风险与 spike 项

| 风险 | spike / 验证 |
|------|-------------|
| Zoom 双轨期间 overlay 与 WS 波形错位 | 2h spike：快速拖动 zoom 滑块，观察 overlay 是否漂移 |
| rAF 播放跟随与 WS `timeupdate` 冲突 | 1h spike：播放中比较 250ms 节流 vs rAF 的 playhead 平滑度 |
| PeakCache Worker 在 Tauri 下路径/transfer 问题 | 4h spike：Worker 加载 `.dat` 并 resample，验证 transfer ownership |
| L1-B 行高 cache stale 导致跳帧 | 1d prototype：filter 切换后行高变化是否引发 layout shift |

## 6. 验收基线

### 自动化

- `npm run typecheck && npm run test && node scripts/check-architecture-guard.mjs` 不回归。

| 薄片 | 验证点 |
