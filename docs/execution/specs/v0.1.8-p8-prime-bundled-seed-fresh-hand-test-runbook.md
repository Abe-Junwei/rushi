## 1. 隔离 Fresh App Data（二选一）

### 方式 A — 脚本隔离 HOME（**推荐**，不污染主 App Data）

```bash
cd /path/to/Rushi
export RUSHI_RELEASE_APP="/Applications/如是我闻.app"   # 改成你的路径

# 先断网（见 §2），再执行：
bash scripts/r3f-fresh-appdata-hand-test.sh --interactive --wipe-ui-prefs
```

脚本会：创建临时 `HOME` → 启动 release `.app` → 自动等 bundled seed → 通过后**保持应用打开**供你继续做 P8′ 目视 / P9′ / P10′。

> **注意**：脚本**不会**替你断网；断网须在 §2 完成后再启动应用。

### 方式 B — 手动 Fresh（从 DMG 安装到干净用户）

1. 使用**从未装过 Rushi** 的 macOS 用户，或先删除主 App Data：
   ```bash
   rm -rf ~/Library/Application\ Support/studio.lingchuang.rushi
   rm -rf ~/Library/WebKit/studio.lingchuang.rushi
   ```
2. 打开 DMG → 拖入「应用程序」。
3. **先断网**（§2）→ 再首次打开 `/Applications/如是我闻.app`。

## 2. 断网（P8′ / P9′ 必做）

任选一种，测 P9′ 前保持断网：

| 方式 | 操作 |
|------|------|
| **Wi‑Fi** | 菜单栏 Wi‑Fi → **关闭** |
| **飞行模式** | 控制中心 → 飞行模式 **开** |
| **验证** | Safari 打开任意网页应失败；或 `ping -c 1 1.1.1.1` 超时 |

> 首启 seed **不应**访问 ModelScope；断网后仍应完成 seed（数据来自安装包内 `bundled-asr-models/`）。

## 3. P8′ — 首启 seed 遮罩（Blocker **B**）

**操作**：Fresh App Data 下**首次**打开 release 包（断网状态）。

**期望（逐项勾选）**：

| ☐ | 检查项 |
|---|--------|
| ☐ | **全屏遮罩**挡住主界面（看不到项目侧栏） |
| ☐ | 文案含 **「正在准备内置语音模型…」**（或 seed 阶段的进度文案） |
| ☐ | 有 **进度条** 且百分比前进（1.1 GB 复制约 **1–5 min**） |
| ☐ | seed 完成后遮罩消失，进入正常主界面 |
| ☐ | **无**「导入离线模型包」类入口 |
| ☐ | **无** ModelScope 下载 / 「准备当前模型」主路径提示 |
| ☐ | 环境页或日志**无** zip 导入相关文案 |

**可选日志佐证**（Fresh HOME 或主 App Data）：

```bash
tail -f ~/Library/Application\ Support/studio.lingchuang.rushi/logs/desktop.log
# 隔离 HOME 时路径见脚本输出的 Fresh HOME
grep -i "bundled\|seed" ~/Library/Application\ Support/studio.lingchuang.rushi/logs/desktop.log | tail -20
```

**Marker 文件**（seed 成功后应存在）：

```bash
ls -la ~/Library/Application\ Support/studio.lingchuang.rushi/models/.rushi-bundled-seed.json
```

**FAIL 时记录**：遮罩文案截图、是否出现 zip/ModelScope UI、`desktop.log` 末 50 行。

## 4. P9′ — 断网本机转写（Blocker **B**）

**前提**：P8′ 已通过，**仍断网**。

| 步 | 操作 | 期望 |
|----|------|------|
| 1 | 新建项目 | 项目创建成功 |
| 2 | 导入 `fixtures/eval/samples/clear.wav`（或自备短音频） | 波形可见 |
| 3 | 转写来源选 **本机 ASR**（若对话框出现） | 非在线 STT |
| 4 | 触发 **自动转录** 或等价本机转写 | 语段出现；无「需联网下载模型」 |
| 5 | 观察顶栏 ASR chip | 本机 ASR 相关状态，非「未就绪」长期挂起 |

**通过标准**：短音频在本机 Paraformer 下产生 ≥1 条语段；进程不 hung；无需联网。

**FAIL 时记录**：`/health` 快照（若 8741 可达）、转写错误 toast、`desktop.log` 中 `transcribe` / `funasr` 关键词。

```bash
curl -sf http://127.0.0.1:8741/health | python3 -m json.tool | egrep 'ready_for_transcribe|funasr_default_model_cached|funasr_required_models_cached'
```

## 7. 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 无遮罩、直接进入主界面 | 非 Fresh（已有 marker） | 删 `models/` 或换隔离 HOME |
| seed 极快（<5s）且 FAIL 转写 | 包内无 `modelscope/` | 重跑 `npm run asr:stage-bundled-models` 再 build |
| 遮罩一直 0% | 磁盘满 / 权限 | 查 `desktop.log`；确认 App Data 可写 |
| P9′ 要求联网 | 走了在线 STT 或未 seed 完 | 确认断网 + `/health` `ready_for_transcribe=true` |
| 仍见 SenseVoice | 旧 WebKit 缓存 | `--wipe-ui-prefs` 或删 WebKit 目录后重测 |
