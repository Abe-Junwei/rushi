# Spec: 本地独立、联机协作、批注审阅与 Word 导出

## 目标
将 Rushi 从“仅支持本地 SQLite 项目库”升级为“本地独立工作 + 联机协作并存”的双层架构，支持多人联机批注、审阅、编辑、记录编辑历史，并保持 Word 导出友好。

额外要求：
- 保留纯本地独立工作模式，不以登录或服务端为前提。
- 协作项目中可看到其他协作者的在线状态、当前编辑位置、编辑进度与最新内容。
- 工作流显式区分“转录模式”和“审阅模式”。
- 只有在审阅模式下显示批注标记、建议修改与审阅相关 UI。

## 背景
当前 Rushi 的真源是桌面端应用数据目录下的 SQLite 与项目音频副本，适合单机编辑与导出，但不适合作为多人协作的正式真源。现有项目包（zip）可作为单项目迁移/归档格式保留，但不承担实时协作职责。

未来产品需要同时覆盖两类工作形态：
- 本地独立项目：单人离线/弱网环境下完成转录、校对、导出。
- 协作项目：多人共同编辑同一项目，并可看到彼此当前编辑内容、进度与审阅状态。

## 受影响代码地图
- apps/desktop/src-tauri/src/db.rs：当前本地 SQLite schema，仅覆盖单机项目、语段与粗粒度 edit_log。
- apps/desktop/src-tauri/src/project/*.rs：当前项目 CRUD、项目包导入导出、ASR 拉取与本地持久化。
- apps/desktop/src/pages/useProjectLifecycleController.ts：当前前端生命周期与本地项目控制入口。
- apps/desktop/src/services/exportFormatters.ts、apps/desktop/src-tauri/src/export_docx.rs：当前 TXT/SRT/DOCX 导出路径。
- 新增服务端（待建）：协作项目、语段、批注、建议修改、历史事件、Presence、导出任务。

## 设计落位
1. docs/architecture/collaboration-storage-schema.md — 协作版数据库 schema 草案。
2. docs/architecture/collaboration-review-domain-api.md — 批注/审阅/Word 导出的领域模型与 API 草案。
3. apps/desktop 本地 SQLite 对本地项目仍为正式真源；在协作项目中退居缓存/离线层。

## 约束
- 不以 zip 项目包承担实时协作真源。
- 不把 Word 导出格式反向当成底层存储格式。
- 首期协作不默认全局 CRDT；先以服务端顺序化操作 + 语段级乐观并发为主。
- 批注、建议修改、编辑历史必须可审计、可导出、可投影到 Word。
- 兼容当前本地桌面能力：离线编辑、项目包导入导出、TXT/SRT/DOCX 导出。
- 本地项目不得因为协作能力引入而失去“无登录即可完整工作”的能力。
- 转录模式默认聚焦正文与时间轴编辑，不显示批注标记。
- 审阅模式才显示批注、建议修改、协作者状态与进度相关 UI。

## 分阶段实施建议
1. P5A：双模式项目模型
   - 明确 `local` 与 `collaborative` 两类项目来源。
   - 本地项目继续使用 SQLite 为真源；协作项目以服务端为真源、本地为缓存。
2. P5B：服务端项目/语段/媒体真源
   - 建立 PostgreSQL + 对象存储 + 项目/语段/版本事件表。
   - 桌面端新增登录、项目列表、加载与保存到服务端能力。
3. P5C：转录模式 / 审阅模式
   - 转录模式聚焦语段正文、时间轴、ASR 与基础编辑。
   - 审阅模式增加评论线程、回复、解决状态、建议修改、接受/拒绝。
4. P5D：Presence、进度与历史
   - 增加协作者 Presence、软锁/占用提示、实时编辑内容与进度视图。
   - 增加 append-only revision_events 与项目活动流。
5. P5E：Word 审阅导出
   - 增加干净稿、批注稿、建议修改附录三类导出投影。
   - 若 DOCX tracked changes 能力不足，先以正文 + 批注/建议附录退化实现。
6. P5F：离线缓存与冲突恢复
   - 协作项目的本地 SQLite 退居缓存/草稿层。
   - 增加 reconnect、冲突提示、离线草稿恢复。

## MVP 范围
- 本地项目与协作项目并存，且入口清晰可区分。
- 服务端项目、语段、批注线程、回复、建议修改、revision_events。
- 桌面端支持多人查看同一协作项目、语段级编辑、评论与建议修改。
- 协作中可见其他成员在线状态、当前编辑位置、已完成/待完成进度与最新内容。
- 转录模式与审阅模式显式切换，批注仅在审阅模式显示。
- Word 导出支持：干净稿、带批注稿、建议修改摘要附录。
- 本地项目包继续支持导入/导出，不强求与所有协作元数据完全对齐。

## 明确不在首期范围
- 全文级别 CRDT。
- 浏览器端完整协作编辑器。
- 复杂权限体系（首期仅项目级 collaborator 角色）。
- Word tracked changes 的完全 OpenXML 等价恢复。

## 验收标准
- [ ] 本地项目可在无登录、无网络条件下完整完成转录、编辑与导出。
- [ ] 协作项目可支持多人协作同一项目，且不依赖本地 SQLite 作为唯一真源。
- [ ] 协作中可见其他成员当前编辑位置、在线状态、进度与最新内容。
- [ ] 语段编辑、批注、建议修改均有结构化持久化与历史记录。
- [ ] 可按时间/人员查看编辑历史与审阅活动。
- [ ] 转录模式不显示批注标记；审阅模式显示批注标记与建议修改。
- [ ] 可导出 Word 干净稿与审阅稿，且批注/建议信息不丢失。
- [ ] 本地项目包仍可作为单项目迁移与归档格式使用。
- [ ] 冲突检测与恢复策略有明确产品口径与技术实现。
