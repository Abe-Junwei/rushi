## §1 — 列表键盘跟手感（LKB · **Blocker**）

> **Plan v2**：[`list-keyboard-navigation-virtual-scroll-performance-plan.md`](./list-keyboard-navigation-virtual-scroll-performance-plan.md) · **§0.1 整树更新矩阵 U1–U20**

| ID | 优先级 | 步骤 | 通过标准 | ☐ P ☐ F | 证据 |
|----|--------|------|----------|---------|------|
| **LKB-H4** | **B** | textarea 内**快速连按** ↓ 20 次后**立刻松手** | 松手后 **≤200ms** 内列表/选中**停止移动**；无持续跳动、无「多跳几段」 | ☑ P | 手测 PASS |
| **LKB-H5** | **B** | 长按 ↓ 至 **OS repeat 停止**，**键仍按住不松** ~3s | **tier / 整页不再**在 ~180ms 后突然滚一次；列表 scrollTop 稳定 | ☑ P | 手测 PASS |
| **LKB-H1** | **B** | **≥500 段**（理想 5000）长按 ↓ 2s | 无空白行槽；选中框跟随；无长段闪白 | ☑ P | 手测 PASS |
| **LKB-H6** | R | 视口中间段 ↑↓ 10 次 | 列表 **minimal scroll**（多数帧不滚）；高亮即时 | ☑ P | |
| **LKB-H7** | R | 快速 ↓ 扫到列表底缘 | 列表 minimal 下滚；选中行不裁切 | ☑ P | |
| **LKB-H8** | R | 松键后 | textarea focus 落到**最终语段**；tier **只 reveal 一次**（focus gate 开时） | ☑ P | |

**LKB v2 代码侧 + Release 候选 `.app` 手测**：H4/H5/H8/H1 **全 PASS**（2026-06-22 签收）。

### LKB 快速脚本（≈12 min）

1. 打开 **≥193 段**项目，焦点在段 0 textarea  
2. **LKB-H4**：↓ 连按 20 次 → 松手 → 观察 200ms 内是否还跳  
3. **LKB-H5**：长按 ↓ 至 repeat 停 → 仍按住 3s → 看 tier 是否偷跳  
4. **LKB-H1/H6**：换 500+ 段项目重复 2–3  
5. **LKB-H8**：松键后确认 focus + tier 只动一次  
6. 可选：`__rushiSelectionProfile.print()` 粘贴 §6 证据
